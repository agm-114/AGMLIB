# Planning workspace

This folder turns repository findings and `TODO.md` into executable,
evidence-backed plans.

## Rules

- Preserve the checkbox source of truth in root `TODO.md`; planning pages expand
  tasks into dependencies, acceptance criteria, risks, and verification.
- Separate current-state evidence from target architecture.
- Every proposed breaking change must include a compatibility and migration
  decision.
- Every runtime plan must identify lifecycle, ownership, multiplayer authority,
  native-version dependencies, and proportional testing.
- File-atlas pages must account for every source file, including legacy,
  experimental, generated, and currently untracked work.

## Map

- `roadmap.md`: phased execution order across the full backlog.
- `build-release-plan.md`, `component-audit-plan.md`, and
  `native-update-plan.md`: detailed plans expanding the root TODO.
- `asset-bundle-serialization-plan.md`: reproduction and research plan for
  custom nested value types that fail across the released bundle-load path.
- `consolidation-plan.md`: target boundaries and extraction proposals.
- `feature-validation-plan.md`: source-derived runtime, multiplayer, save/load,
  and pooling validation matrices for the current gameplay feature set.
- `risk-register.md`: current repository and release risks.
- `patterns/agm.md`: AGMLIB patterns shaped by external modding, ad-hoc modder
  needs, composition, typed adapters, and removable sidecars.
- `patterns/lys.md`: native-source-only analysis of LYS's planned, inheritance-
  heavy, authoritative mega-class architecture.
- `patterns/AGENTS.md`: attribution rule separating AGM, LYS, and other
  repository contributors/modders.
- `generated/`: reproducible inventories generated by
  `scripts/Documentation/Export-AgmlibInventory.ps1` and the exhaustive 233-item
  plan generated by `Export-TodoPlan.ps1`.
