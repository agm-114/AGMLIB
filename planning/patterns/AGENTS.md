# Pattern-analysis boundaries

- `agm.md` describes patterns in AGMLIB source plus maintainer-provided context
  about supporting modders.
- `lys.md` describes patterns in the pinned native NEBULOUS source only.
- Never infer that a Git author, contributor, or other modder is LYS.
- Treat non-AGM repository committers as other modders/contributors unless the
  maintainer explicitly identifies them differently.
- Other-modder changes may be discussed as individual contributions, but they
  are not evidence for AGM or LYS implementation patterns.
- Keep identity claims out of architectural analysis unless the maintainer has
  supplied the identity explicitly.
